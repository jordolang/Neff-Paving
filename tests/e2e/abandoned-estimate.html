🧪 End-to-End Test: Abandoned Estimate Flow

This test validates the complete abandoned estimate nurture flow from estimate creation to follow-up email.

Test Overview

Purpose: Verify that abandoned estimates trigger automated follow-up emails

Steps:

  1. Create estimate via form with email consent checked
  2. Verify lead stored in localStorage with status='new'
  3. Manually trigger check-abandoned API (simulate cron)
  4. Verify email API called with abandoned_estimate template
  5. Verify lead status updated to 'contacted'
  6. Verify analytics event tracked

Configuration

1Create Estimate with Email Consent

Simulates user creating an estimate and opting in to email communications.

2Verify Lead Stored in localStorage

Checks that the lead was correctly saved with status='new' and email consent.

3Trigger check-abandoned API

Manually calls the cron endpoint to check for abandoned estimates (simulates scheduled task).

4Verify Email API Called

Confirms that the abandoned_estimate email was requested.

5Verify Lead Status Updated

Checks that the lead status was updated from 'new' to 'contacted'.

6Verify Analytics Event Tracked

Confirms that analytics events were recorded for the nurture campaign.
